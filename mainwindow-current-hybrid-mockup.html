This mockup keeps your actual MainWindow architecture intact: Menu, MainToolbarControl, NavSidebarControl, ItemListPanelControl, splitter, and DetailPaneHeaderControl. The visual shift is mostly a shell and styling pass, so Avalonia is not the blocker; the real lift is normalizing dozens of detail panels and shared styles once the shell direction is chosen.
What is straightforward in Avalonia
Menu + toolbar rows Left nav bar Status bar segments Grid splitter layout DataGrid density tuning
What makes it a significant project
  1. The shell is easy; making every detail panel feel consistent is the bigger effort.
  2. Your current Suki-based tokens would need either a hybrid theme layer or a stronger custom visual system.
  3. Density, spacing, typography, and header patterns would need to be rolled out across many reusable controls.
